# Lab 2: Timer 测试说明

本实验测试 `pa_perips_timer` 的寄存器接口、计数过程、禁用行为、重启行为和中断输出。测试台会通过总线写入定时器寄存器，观察 `COUNT` 的装载和递减，等待 `irq_o` 触发，并检查 `SR` 的中断标志行为。

## 运行

```bash
make all
make wave
make clean
```

`make all` 会编译并运行测试。`make wave` 会生成 `timer.vcd`，并用 `timer.gtkw` 打开 GTKWave。`make clean` 清理仿真产物。

## 寄存器约定

测试台使用以下地址访问定时器：

- `0x00`: `CR`，控制寄存器，`CR[0]` 用于启动/停止定时器
- `0x04`: `SR`，状态寄存器，测试中用于观察和清除中断状态
- `0x08`: `PSC`，预分频寄存器
- `0x0c`: `LOAD`，计数加载值
- `0x10`: `COUNT`，当前计数值

## 测试内容

### 1. Reset Defaults

复位后依次读取 `CR`、`SR`、`PSC`、`LOAD`、`COUNT`，并检查 `irq_o`。

期望行为：

- `CR = 0`
- `SR = 0`
- `PSC = 0`
- `LOAD = 0`
- `COUNT = 0`
- `irq_o = 0`

这个测试能发现寄存器没有复位、复位值写错、或者 `irq_o` 默认拉高的问题。

### 2. Register Readback While Disabled

测试流程：

1. 写 `PSC = 2`
2. 写 `LOAD = 7`
3. 写 `CR = 0` 保持定时器禁用
4. 读回 `PSC`、`LOAD`、`CR`
5. 等待若干周期，确认不会产生中断
6. 读取 `COUNT`

期望行为：

- `PSC` 和 `LOAD` 能正确保存并读回
- `CR[0] = 0` 时定时器不运行
- 禁用状态下 `irq_o` 保持 0
- 禁用状态下 `COUNT` 保持 0

这个测试能发现寄存器地址译码错误、禁用状态仍然计数、或者写 `PSC/LOAD` 后无法读回的问题。

### 3. Enable, Load, Count Down, IRQ Pulse

测试流程：

1. 写 `PSC = 1`
2. 写 `LOAD = 4`
3. 写 `CR = 1` 启动定时器
4. 读回 `CR`
5. 读取 `COUNT`，确认启动后已经从 `LOAD` 装载
6. 等待若干周期后再次读取 `COUNT`，确认计数器在递减
7. 等待 `irq_o` 触发
8. 读取 `SR`，确认能看到中断标志
9. 确认 `irq_o` 随后自动回落

期望行为：

- 写 `CR[0]=1` 后，`CR` 能读回使能状态
- `COUNT` 会从 `LOAD` 装载
- 随着时钟推进，`COUNT` 会按预分频节奏递减
- 计数结束后 `irq_o` 拉高
- `SR[0]` 能记录中断标志
- 本设计中 `irq_o` 是短脉冲，读到 `SR[0]` 后 `irq_o` 可能已经回落

注意：读取 `COUNT` 本身也会消耗时钟周期，所以测试检查的是经过若干读操作和等待后的确定值，而不是“刚启动后的瞬时值”。

### 4. Clear Register Write And Restart

测试流程：

1. 向 `SR` 写 0，确认中断标志保持清除
2. 写 `CR = 0` 禁用定时器
3. 等一个周期后读取 `COUNT`
4. 写入新的 `LOAD = 2` 和 `PSC = 0`
5. 再次写 `CR = 1` 启动
6. 等待新的中断触发

期望行为：

- 写 `SR=0` 不应错误地产生新的中断
- 禁用定时器后，`COUNT` 会清零
- 修改 `LOAD/PSC` 后重新启动，定时器应使用新参数
- 旧一轮中断状态不能影响新一轮运行

这个测试能发现重启没有重新加载参数、禁用时没有清理计数器、或者中断清除逻辑不稳定的问题。

### 5. Unknown Address And Disable Stability

测试流程：

1. 禁用定时器
2. 向未知地址 `0xfc` 写入 `0xffff_ffff`
3. 读取未知地址
4. 等待若干周期，确认没有中断

期望行为：

- 未知地址读回 `0`
- 未知地址写入不应破坏定时器状态
- 禁用状态下继续保持无中断

这个测试能发现默认分支没有处理、错误地址污染寄存器、或者禁用后仍然有残留 IRQ 的问题。

## 失败时看什么

终端会打印当前总线地址、读写使能、写数据、读数据和 `irq_o`。调试时优先在波形中查看：

- `data_we_i` 有效时，目标寄存器是否被写入
- `CR[0]` 是否真正启动或停止计数
- `PSC` 分频计数是否清零、递增和比较正确
- `LOAD` 是否在启动或重新装载时进入 `COUNT`
- `COUNT` 是否按预期递减或归零
- `SR` 和 `irq_o` 的置位/清除关系是否正确

常见错误：

- 写 `CR` 后没有重新加载 `COUNT`
- `PSC=0` 处理成永远不计数
- 中断触发后 `irq_o` 一直不清除
- 读寄存器时 `data_o` 地址选择错误
- 禁用定时器时忘记清理计数状态
